## 三、盈亏分析

### 以 ¥80,000 目标为例

| 项目 | 金额 |
|------|------|
| 众筹总额 | ¥80,000 |
| 摩点平台费(6%) | -¥4,800 |
| 支付手续费(~1%) | -¥800 |
| **实际到账** | **¥74,400** |
| | |
| 固定成本 | -¥80,000 |
| 硬件成本(250台×¥140) | -¥35,000 |
| **总成本** | **-¥115,000** |
| | |
| **盈亏** | **-¥40,600** |

**结论**：首批众筹很难盈利，主要目的是验证市场和筹集启动资金。

### 盈亏平衡点

要实现盈亏平衡，需要：
- 假设固定成本 ¥80,000
- 单台硬件成本 ¥140
- 平均售价 ¥320
- 平台费率 7%

**计算**：
```
盈亏平衡销量 = 固定成本 / (平均售价×0.93 - 硬件成本)
             = 80,000 / (320×0.93 - 140)
             = 80,000 / (297.6 - 140)
             = 80,000 / 157.6
             ≈ 508 台
```

**需要卖出约 508 台才能盈亏平衡。**

## 四、现金流预测

摩点采用 5-3-2 结算模式：
- 首款：50%（众筹成功后）
- 中款：30%（全部发货后）
- 尾款：20%（支持者评价期后）

### 以 ¥80,000 目标为例

| 阶段 | 时间 | 金额 | 用途 |
|------|------|------|------|
| 首款 | T+5工作日 | ¥37,200 | 启动生产 |
| 中款 | T+90天 | ¥22,320 | 覆盖生产尾款 |
| 尾款 | T+120天 | ¥14,880 | 运营储备 |

## 五、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 众筹未达标 | 资金退回，项目延期 | 设置保守目标（¥50,000），确保达成 |
| 成本超支 | 亏损扩大 | 严格控制BOM，预留15%缓冲 |
| 发货延期 | 用户退款 | 保守承诺（90天），实际争取60天发货 |
| 质量问题 | 退款+口碑受损 | 加强测试，预留5%备品 |
