## 一、我们看到的问题

### AI 智能体被困在电脑里

2025-2026 年，AI 智能体（Agent）的能力飞速发展。它们能写代码、发邮件、查信息、管理日程。但有一个根本性的缺陷：

> **你的 AI 智能体只在你主动打字的时候才知道你在做什么。**

它没有眼睛，看不到你每天截图的信息。它没有耳朵，听不到你在开车时冒出的想法。它被困在聊天窗口里，等着你一个字一个字地打出指令。

这就像雇了一个能力极强的助理，但他被锁在一间没有窗户的房间里——你必须走进去、敲门、写一封信塞进门缝，他才能开始工作。

**问题的本质不是 AI 不够聪明，而是 AI 缺少感知你日常生活的通道。**

### 用户面临的真实困境

| 场景 | 用户的痛苦 | 摩擦有多大 |
|------|-----------|-----------|
| 开车时想到一件事 | 不能掏手机，想法 5 分钟后就忘了 | 无法捕获 |
| 微信群里看到有用信息 | 截图存下来，但之后找不到了 | 截完就忘 |
| 想让 AI 帮忙发邮件 | 解锁手机 → 打开 App → 等加载 → 打字 → 确认 | 7 步，45 秒+ |
| 散步时灵感爆发 | 没有快速捕获机制，只能努力记住 | 想法流失 |
| 会议中截了需求变更 | 截图躺在相册里，没人提醒跟进 | 信息沉睡 |

**这些场景的共同特征：用户的 AI 智能体本可以帮忙，但它感知不到这些信息。**

### 给 AI 智能体装上"眼睛"和"耳朵"

EinClaw 正在构建一套 **双通道感知系统**，让用户的 AI 智能体能够持续感知日常生活：

## 三、产品一：EINKO — 视觉通道

### 一句话定位

**EINKO 是你的视觉记忆代理 — 截图、拍照，然后忘掉它。**

### 它解决什么问题

用户每天截图 5-15 张：聊天记录里的地址、朋友推荐的餐厅、工作群的需求变更、社交媒体上的灵感。这些截图的共同命运是 **截完就忘，需要时找不到**。

EINKO 改变了这个局面：

| 传统截图 | EINKO |
|---------|-------|
| 截图 = 一张死图片 | 截图 = 被 AI 理解的活信息 |
| 找图 = 按时间线盲翻 | 找图 = 自然语言搜索 "上周那个日料餐厅" |
| 截完没人管 | AI 自动提取地址、日期、待办，主动提醒 |
| 手动整理分类 | AI 自动分类，零整理成本 |

### 核心功能（已上线）

- **截图自动导入**：监测相册新增截图，无感知导入
- **AI 内容理解**：自动识别内容类型（聊天、网页、代码、地图等）
- **智能分类**：AI 根据内容自动归类
- **语义搜索**：通过关键词或自然语言搜索截图内容
- **手动拍照导入**：支持主动拍照捕获现实场景

### 当前进展

| 指标 | 数据 |
|------|------|
| Beta 用户数 | **449 人** |
| 产品状态 | 已上架 App Store |
| 核心功能 | 全部上线可用 |

### 发展路线

- **Phase 1（当前）**：验证"AI 自动理解截图"的核心价值
- **Phase 2**：从"理解"到"行动"——信息提取、智能提醒、一键操作
- **Phase 3**：与 OpenClaw 智能体深度联动，跨通道协作

## 四、产品二：EinClaw 对讲机 — 听觉通道

### 一句话定位

**EinClaw 对讲机是你和 AI 智能体之间的直线电话 — 按下按钮，说出指令，松手走人。**

### 核心比喻：Use your agent like a special agent

想象一个特种兵在战场上执行任务。他不是一个人在战斗——他身后有整个指挥中心。他通过对讲机和后方保持联系：

| 特种兵场景 | EinClaw 用户场景 |
|------------|----------------|
| 通过对讲机呼叫后方 | 按下按钮对 AI 说话 |
| 指挥中心接收并协调资源 | AI Agent 接收指令并调度工具 |
| 火炮/空中支援执行打击 | Agent 执行任务 |
| 特种兵只关心：目标是否被打掉 | 用户只关心：事情是否被办好 |

**一个按钮，一个麦克风，一条到 Agent 的直线。就够了。**

### 为什么是"对讲机"而不是"麦克风"

| 维度 | 麦克风（被动录音） | 对讲机（主动对话） |
|------|-------------------|-------------------|
| 用户心智 | "它在记录我" | "我在和它说话" |
| 隐私感知 | "它一直在听" — 有压力 | "我主动按才说" — 我掌控 |
| 交互意图 | 捕获环境信息 | 下达指令、分配任务 |
| 反馈机制 | 无即时反馈 | 振动/LED 确认已接收 |

### 用户旅程

```
开车上班，想到今天要做的事
    ↓
按下夹在衣领上的对讲机按钮（红灯 + 短振动）
    ↓
"帮我给张三发邮件，告诉他周五方案评审改到下周一"
    ↓
松开按钮（绿灯闪 + 双短振 = 已上传）
    ↓
继续开车，不再想这件事
    ↓
到办公室打开电脑，Agent 已起草好邮件，一键发送
```

**从想法到行动的路径：3 步，5 秒，单手操作，无需看屏幕。**

### 硬件规格

| 组件 | 规格 | 成本 |
|------|------|------|
| 主控芯片 | ESP32-S3-WROOM-1 | ~$3.50 |
| 麦克风 | INMP441 (I2S 数字) | ~$1.50 |
| MicroSD 模块 | SPI 接口 | ~$0.80 |
| 锂电池 | 3000mAh | ~$3.00 |
| 其他（按钮/LED/振动马达/PCB） | — | ~$3.30 |
| **BOM 合计** | | **~$12.10** |

- **形态**：夹子式，约 AirPods 充电盒一半大小
- **续航**：混合使用预估 3-5 天
- **连接**：WiFi（手机热点 / 家庭 WiFi）

### 开发阶段

- **Phase 1（4-6 周）**：开发板原型验证，打通端到端链路
- **Phase 2（8-12 周）**：功能完善 + 可佩戴化 + 20-50 名 Beta 用户
- **Phase 3（16-24 周）**：定制 PCB + 工业设计 + 开模 + 量产

## 五、技术架构

### 全链路数据流

```
  EINKO (视觉)                       对讲机 (语音)
  截图/照片                           按下按钮说话
      |                                   |
      v                                   v
  AI 理解 + 分类                    WAV 编码 + WiFi 上传
      |                                   |
      v                                   v
  ┌──────────────────────────────────────────┐
  │            OpenClaw 智能体平台            │
  │                                          │
  │  · STT 语音转文字（Whisper 多级降级）     │
  │  · 多模态内容理解                         │
  │  · 任务规划与执行                         │
  │  · 长期记忆系统                           │
  │  · 工具调用（邮件/日历/搜索/文件...）     │
  └──────────────────────────────────────────┘
                      |
                      v
              Agent 执行结果
     （邮件已发 / 提醒已设 / 信息已查）
```

### 安全架构

| 层级 | 保护措施 |
|------|----------|
| 设备 → 服务器 | HTTPS 加密传输 + API Key |
| 服务器 → OpenClaw | localhost 调用 + Bearer Token |
| 音频数据 | STT 转写完成后即删原始音频 |
| 截图数据 | 端侧优先处理，仅元数据上云 |

### 为什么是现在

1. **AI Agent 能力刚刚成熟**：GPT-4、Claude 等模型让 Agent 真正能执行复杂任务
2. **硬件成本极低**：ESP32-S3 + 数字麦克风 + 锂电池，BOM 仅 $12
3. **竞品已验证需求但方向错误**：Humane AI Pin ($699) 和 Rabbit R1 ($199) 证明了用户想要 AI 硬件，但价格和复杂度都太高
4. **中国供应链优势**：深圳供应链让我们能以极低成本快速迭代硬件
