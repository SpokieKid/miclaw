硬件详情 · 为什么坚持 4G 独立

4G 独立联网 — 让硬件真正脱离手机

"说出来,就完成了" 的体验承诺,前提是 用户什么都不用想就能开口。一旦还要先掏手机、连蓝牙、开 App,这个承诺就崩了。
理由 1 · 心智零负担

用户不需要"想起来怎么用"

没有蓝牙配对、没有手机 App、没有 Wi-Fi 配置。一颗按键,按下就能说话。
一旦输入入口要先确认 "手机有没有连上", "说出来,就完成了" 这个承诺就崩了 — 用户最终还是会回到掏手机的旧动作。
理由 2 · 入口独占

硬件是入口,不是手机外设

靠蓝牙或 Wi-Fi 接手机的硬件,本质上只是手机的一个外设 — 入口仍在手机厂商手里。
4G 独立联网,意味着 "输入这一刻" 的决策权第一次真正归硬件所有 — 这是构建独立 AI 入口的硬件前提。
4G
独立联网 + 独立存储
不依赖手机蓝牙、不依赖 Wi-Fi · 出门即在线
7 天
日常续航
日常 7 天充一次 · 持续录音可达 150 分钟
1 键
极简交互

设计原则
硬件只做语音输入 — 不做语音输出、不做结果展示。 所有 "看结果" 的需求交给 Notion / 飞书 / X / Email 等用户已经在用的终端。

产品矩阵

硬件不是孤岛 — 入口 × 云端的双轮闭环

硬件 = 输入入口 · 云端 OpenClaw Agent 平台 = 执行 + 跨系统交付。 两端缺一,闭环就断。没有 4G 独立,硬件就只是手机外设 — 入口被手机厂商持有,Agent 调度被 App 范式拖慢,我们就回到了 Plaud / 录音豆的老问题。 4G 独立让我们第一次真正拥有输入这一刻;云端则把结果交付到 Notion / 飞书 / X / Email 等用户既有终端,不抢用户已经习惯的"看结果"心智 — 这才是壁垒。
Path 1 · Hardware-only

仅购买 Einclaw Mic

已有 OpenClaw / 自建 Agent 工作流的 power user,直接接 API。
收入:硬件单价
Path 2 · Cloud-only

仅订阅 Einclaw 云端

通过 Telegram / Discord / 桌面端调用 OpenClaw,先体验 skill 价值。
收入:月度订阅
Path 3 · Bundle (主推)

硬件 + 云端打包

完整 Voice-to-Action 体验,丝滑零摩擦 — 主推产品线。
收入:硬件 + 订阅 + Skill 分成 (LTV 最高)
OpenClaw 平台
开放 Agent 分发层
